# Particle Draw

A browser-based generative art tool that spawns noise-driven particles from a moving origin, producing organic clouds, trails, and fractal dust formations in real time.

Open `particle_draw.html` directly in any modern browser — no build step or server required.

---

## Overview

Each frame a new particle is deposited at the current spawn position. The particle drifts under an acceleration field generated by one of six noise algorithms, loses alpha according to its decay rate, and — when it fades to 25% of its initial opacity — recursively spawns two smaller children. The children can recurse further, creating self-similar particle clouds from a single source. A second mirrored spawn head can be enabled for bilateral symmetry.

All parameters update live. The canvas never stops rendering; changes take effect on newly spawned particles immediately.

## Interface

The UI is split into a **canvas area** (left) and a **parameter panel** (right, 320 px wide).

### Canvas toolbar

Buttons float at the bottom of the canvas:

| Button | Action |
|---|---|
| Pause / Resume | Halt or continue particle spawning |
| Clear | Remove all particles and reset the canvas to black |
| Show Spawn / Hide Spawn | Toggle the animated crosshair that marks the spawn point |
| Export PNG | Download the current canvas as a PNG |
| Record / Stop | Start or stop WebM video capture of the canvas at 60 fps |
| Manual / Canvas | Toggle the built-in user manual overlay |

### Panel tabs

| Tab | Contents |
|---|---|
| Spawn | Position, motion, decay, presets |
| Noise | Noise type selector and strength |
| Color | Hue, saturation, alpha, particle size |
| Mod | LFO oscillators for automated parameter modulation |
| Info | Quick reference and shortcut list |

## Parameters

### Spawn tab — Position

**Spawn Point mini-map**
Click anywhere on the mini-map at the top of the Spawn tab to place the draw origin. The yellow crosshair indicates the primary head; the cyan crosshair (Mirror mode only) shows the reflected second head.

**Draw X / Draw Y** `0–1`
Normalized canvas coordinates for the spawn origin. X = 0 is the left edge, Y = 0 is the bottom edge. Changing either resets the motion accumulator so the origin snaps cleanly to the new position.

**Draw Heads**
- `1 — Single` one spawn origin.
- `2 — Mirror` a second origin reflected across the canvas center, producing bilateral symmetry. Also raises the recursion threshold from 20 to 30.

### Spawn tab — Motion

**Speed** `0–1`
How far the spawn origin moves each frame along the chosen direction. At 0 the origin is stationary; at 1 it traverses the canvas quickly, leaving a moving streak.

**Direction** `0–1`
Travel angle of the spawn origin (0 = 0°, 1 = 360°). The compass widget in the panel shows the current heading. Modulate with an LFO for sweeping arcs or full circular paths.

### Spawn tab — Decay

**Decay Rate** `2–20`
Alpha lost per frame. The primary control over particle lifetime.
- Low (2–4): long-lived clouds that accumulate and overlap.
- High (10–20): brief sparks that vanish almost immediately.

**Trail Fade** `0–0.4`
A semi-transparent black rectangle drawn over the entire canvas each frame. At 0 nothing smears. At higher values only the most recent particles are fully visible, creating comet-like motion blur. Works best with a fast-moving spawn point and a high decay rate.

---

### Noise tab — Noise Type

The noise type determines the acceleration vector applied to each particle every frame.

| # | Type | Character |
|---|---|---|
| 1 | **Perlin** | Position-based gradient noise. Smooth, coherent flow fields. The best default for organic results. |
| 2 | **Random** | Pure uniform randomness. Explosive, jittery scatter with no spatial coherence. |
| 3 | **Gaussian** | Normal-distribution randomness. Symmetric soft diffusion — most particles stay close to the spawn point. |
| 4 | **fBm** | Fractal Brownian Motion — layered octaves of value noise. Multi-scale turbulence with both large sweeps and fine detail. |
| 5 | **Brownian** | Persistent random walk. Each particle accumulates a velocity direction and drifts continuously, producing flowing river-like motion. |
| 6 | **Logistic** | Deterministic chaos via the logistic map (r = 3.9). Fully reproducible yet apparently random; creates dense, unpredictable tangles. |

Select a noise type by clicking its card in the panel or pressing keys `1`–`6`.

**Noise Strength** `0.01–3`
Scales the acceleration vector the noise applies each frame. Higher values spread particles more aggressively; near 0 they drift gently around the spawn point.

---

### Color tab

**Hue** `0–1` → `0°–360°`
Base hue for all newly spawned particles. Press `R` for a random value.

**Saturation** `0–1`
Color saturation. 0 = greyscale, 1 = fully saturated. The color strip at the top of the tab previews the current hue + saturation.

**Alpha Multiplier** `0–1`
Scales the drawn opacity of every particle. Lower values produce a transparent, layered wash that accumulates with overlap.

**Particle Size** `1–20 px`
Width and height of each square particle. Children spawned via recursion inherit this size.

## Recursion

When a particle's alpha drops below 25% of its initial value — and that initial value exceeds the recursion threshold — it spawns two children at its current position. Children inherit half the parent's decay rate, so they live roughly twice as long and can recurse again. The cascade continues until the alpha is too faint to recurse further.

| Mode | Recursion threshold |
|---|---|
| Single head | 20 |
| Mirror (2 heads) | 30 |

This produces fractal dust clouds automatically without any manual intervention. The effect is most visible with Perlin or fBm noise at a medium decay rate (4–8).

## LFO Modulation

The Mod tab provides a bank of oscillators that automatically sweep any panel parameter between a Lo and Hi value on a repeating cycle, enabling animated patterns without manual slider movement.

### Adding an oscillator

1. Go to the **Mod** tab and click **+ LFO**.
2. Inside the new LFO card, click **+ map** and select a target parameter from the dropdown.
3. Set the **Lo** and **Hi** bounds for the sweep range.
4. Adjust **Rate**, **Amp**, and **Phase** to taste.

Modulated sliders turn **cyan** in the panel and display the live driven value next to the static control value.

### LFO parameters

| Parameter | Range | Description |
|---|---|---|
| Shape | — | Waveform: `SIN` smooth · `TRI` linear ramp · `SQR` hard on/off · `SAW` rising · `RSW` falling |
| Rate | 0.01–10 Hz | Cycles per second. 0.1–1 Hz is a good starting range for slow evolution. |
| Amp | 0–1 | Scales the bipolar contribution before mapping. At 0 the LFO has no effect. |
| Phase | 0–1 | Cycle starting offset. Shift multiple LFOs out of phase for complex compound motion. |

### Multiple LFOs on one parameter

When more than one LFO targets the same parameter their contributions are **averaged**. This blends their influence rather than stacking it, keeping values within the defined Lo/Hi bounds.

### Modulatable parameters

Decay Rate · Trail Fade · Hue · Saturation · Alpha Multiplier · Particle Size · Draw X · Draw Y · Speed · Direction · Noise Strength

## Presets

Six built-in presets on the Spawn tab demonstrate distinct visual styles:

| Preset | Character |
|---|---|
| **Diffuse** | Gentle Perlin drift, single head, no trail |
| **Flow** | Slow directional flow with a light trail |
| **Chaos** | Fast logistic noise, mirrored heads, aggressive scatter |
| **Bloom** | Low decay Gaussian diffusion, soft expanding cloud |
| **Trails** | Brownian motion with a strong trail fade, long comet tails |
| **Storm** | High decay random noise, dual heads, heavy smearing |

Clicking a preset applies all its parameters, clears the canvas, and resets the spawn accumulator.

---

## Output

| Format | How |
|---|---|
| PNG still | Click **Export PNG** — downloads the current canvas frame |
| WebM video | Click **Record** to start, click **Stop** to finish and download |

Video is captured directly from the canvas element at 60 fps using the `MediaRecorder` API with VP9 encoding where available. The output contains only the canvas — the parameter panel is not included in the recording.

---

## Browser compatibility

Requires a browser with Canvas 2D, `MediaRecorder`, and `canvas.captureStream()` support. All modern versions of Chrome, Edge, Firefox, and Safari work. For best recording compatibility, Chrome or Edge is recommended (broadest VP9 support).
