Particle Draw
User Manual — Parameter Studio
Overview

Particle Draw spawns small squares at a moving origin each frame. Each particle drifts under a noise-driven acceleration field, fades out according to its decay rate, and can leave a persistent trail. Particles also recursively spawn smaller children before dying, producing fractal-like cloud formations from a single source.

The canvas renders continuously. All parameter changes affect newly spawned particles immediately. Use Presets on the Spawn tab for quick starting points. Export a still with Export PNG or capture a WebM video with Record.

Spawn & Position Spawn tab
Spawn Point mini-mapClick anywhere on the mini-map at the top of the Spawn tab to set the draw origin directly. The yellow crosshair shows the primary head; the cyan crosshair (Mirror mode) shows the reflected second head.
Draw X / Draw YNormalized (0–1) spawn coordinates — X=0 is the left edge, Y=0 is the bottom edge. Changing either resets the motion accumulator so the origin snaps to the new position.
Draw Heads1 — Single: one spawn origin. 2 — Mirror: a second origin reflected across the canvas center, producing bilateral symmetry.
SpeedHow fast the spawn origin moves each frame in the chosen Direction. At 0 the origin is stationary; at 1 it crosses the canvas quickly, leaving a moving streak.
DirectionTravel angle of the spawn origin (0–1 = 0°–360°). The compass in the panel shows the current heading. Modulate with an LFO for sweeping arcs or circular paths.
Decay & Trails Spawn tab
Decay RateAlpha lost per frame. Higher values = short-lived sparks; lower values = long, persistent clouds. Range 2–20. This is the primary control over particle lifetime.
Trail FadeA semi-transparent black rectangle is drawn over the canvas each frame. At 0 nothing fades. At higher values only very recent particles stay visible, creating comet-like motion blur. Best combined with a fast-moving spawn point.
Noise Types Noise tab
PerlinPosition-based gradient noise. Particles follow coherent, smoothly curving paths. The best default for organic, flowing results.
RandomPure uniform randomness each frame. Produces explosive, jittery scatter with no spatial coherence. Good for sparks or static.
GaussianNormal (bell-curve) distribution. Symmetric soft diffusion — most particles stay close to their spawn point, forming round clouds.
fBmFractal Brownian Motion — layered octaves of value noise. Produces multi-scale turbulence with both large sweeping motions and fine-grain detail.
BrownianPersistent random walk. Each particle accumulates a velocity direction and drifts continuously in it, giving a flowing river-like quality that gradually spreads.
LogisticDeterministic chaos via the logistic map (r = 3.9). Fully reproducible yet apparently random — creates dense, unpredictable tangles that never repeat.
Noise StrengthScales the acceleration vector the noise applies each frame. Higher values produce more turbulent, spread-out clouds; near 0 particles drift very gently.
Recursion

When a particle's alpha drops below 25% of its initial value — and that initial value exceeds the recursion threshold (20 for single head, 30 for mirror mode) — it spawns two smaller children at its current position. The children inherit half the parent's decay rate, so they live roughly twice as long and can recurse again. This cascades until particles are too faint to recurse further, producing fractal dust clouds from a single source particle.

Color & Shape Color tab
HueBase hue (0–1 maps to 0°–360°) used for all newly spawned particles. Press R to jump to a random hue instantly.
SaturationColor saturation (0 = greyscale, 1 = fully saturated). The color strip at the top of the Color tab previews the current hue + saturation combination.
Alpha MultiplierScales the computed alpha of each particle when drawn. Lower values produce a more transparent, layered watercolour wash.
Particle SizeWidth and height of each square particle in pixels (1–20). Children spawned via recursion inherit the current particle size.
LFO Modulation Mod tab

Oscillators (LFOs) automatically sweep any parameter between a Lo and Hi value on a repeating cycle. Add one with + LFO, then click + map inside the card to assign it to a parameter. Active modulation turns sliders cyan and shows the live driven value.

ShapeWaveform — SIN (smooth), TRI (linear ramp), SQR (hard on/off), SAW (rising), RSW (falling).
RateCycles per second (0.01–10 Hz). At 0.01 Hz one full sweep takes 100 s; at 10 Hz it pulses near-audibly. Good starting range: 0.1–1 Hz.
AmpScales the bipolar contribution before mapping. At 0 the LFO has no effect regardless of its Lo/Hi range.
PhaseCycle starting offset (0–1). Shift multiple LFOs out of phase for complex compound motion patterns.
Lo / Hi rangeEach mapping defines independent sweep bounds for its parameter. Multiple LFOs targeting the same parameter are averaged together.
